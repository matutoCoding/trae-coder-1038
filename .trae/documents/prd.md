## 1. 产品概述

电梯维保企业作业管理APP——面向电梯维保企业的移动端作业管理平台，为维保工提供电梯台账管理、维保派工、保养打卡、故障报修、困人救援等全流程数字化支持，解决传统纸质工单效率低、响应慢、数据难追溯的痛点，助力企业实现维保作业规范化、救援响应高效化、绩效管理数据化。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 维保工 | 管理员分配账号 | 接收派工、到场打卡、完成保养、故障报修、参与救援 |
| 调度员 | 管理员分配账号 | 派工调度、救援指挥、工单跟踪 |
| 管理员 | 系统预设 | 全部权限、人员管理、合同管理、绩效查看、系统设置 |

### 2.2 功能模块

1. **电梯台账**：电梯设备台账、维保合同到期提醒、电梯停用挂牌
2. **维保派工**：半月保季度保派工、工单状态跟踪、维保人员持证管理
3. **保养打卡**：扫码到场打卡、保养项目逐项确认、零部件更换记录
4. **故障报修**：故障报修工单创建、故障分类与图片上传、处理进度跟踪
5. **困人救援**：电梯困人救援响应、应急到场计时、救援过程记录
6. **年检管理**：年检定检排期、年检结果录入、超期预警
7. **绩效统计**：维保完成率统计、救援响应时间分析、人员工作量排名

## 3. 核心流程

### 3.1 维保派工→保养打卡流程
调度员根据半月保/季度保计划创建派工单，系统自动匹配持证人员；维保工收到派工通知后接单，到场扫描电梯二维码签到打卡，逐项完成保养项目并拍照确认，如需更换零部件则登记更换记录，全部完成后提交保养报告。

```mermaid
flowchart TD
    "调度员创建派工单" --> "系统匹配持证人员"
    "系统匹配持证人员" --> "维保工接收派工通知"
    "维保工接收派工通知" --> "扫码到场签到"
    "扫码到场签到" --> "逐项完成保养项目"
    "逐项完成保养项目" --> "是否更换零部件"
    "是否更换零部件" -- "是" --> "登记零部件更换记录"
    "是否更换零部件" -- "否" --> "提交保养报告"
    "登记零部件更换记录" --> "提交保养报告"
    "提交保养报告" --> "调度员审核确认"
```

### 3.2 困人救援流程
接到困人报警后系统自动创建救援工单，调度员指派最近维保工，系统开始应急到场计时；维保工赶赴现场到场打卡停止计时，执行救援操作，完成后记录救援过程和原因分析。

```mermaid
flowchart TD
    "困人报警接入" --> "系统创建救援工单"
    "系统创建救援工单" --> "调度员指派维保工"
    "调度员指派维保工" --> "应急到场计时开始"
    "应急到场计时开始" --> "维保工赶赴现场"
    "维保工赶赴现场" --> "到场打卡停止计时"
    "到场打卡停止计时" --> "执行救援操作"
    "执行救援操作" --> "记录救援过程与原因"
    "记录救援过程与原因" --> "救援工单关闭"
```

### 3.3 故障报修流程
维保工或物业发现故障后创建报修工单，上传故障图片和描述，调度员分配维修人员，维修人员到场维修并记录处理过程，完成后提交维修报告。

```mermaid
flowchart TD
    "发现电梯故障" --> "创建报修工单"
    "创建报修工单" --> "上传故障图片与描述"
    "上传故障图片与描述" --> "调度员分配维修人员"
    "调度员分配维修人员" --> "维修人员到场维修"
    "维修人员到场维修" --> "记录处理过程"
    "记录处理过程" --> "提交维修报告"
    "提交维修报告" --> "工单关闭"
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：工业蓝 #1B5E8C（专业、可靠），辅以安全橙 #E8721C（紧急、预警）
- **辅助色**：状态绿 #2E9E5A（完成/正常）、告警红 #D94452（超期/故障）、中性灰 #6B7280
- **背景色**：浅灰 #F5F6FA，卡片白 #FFFFFF
- **按钮风格**：圆角6px，主按钮填充色，次要按钮描边风格
- **字体**：标题用思源黑体/Noto Sans SC Bold，正文用Noto Sans SC Regular
- **布局风格**：移动端底部Tab导航，卡片式信息展示，列表与详情二级结构
- **图标风格**：线性图标，2px线宽，圆角端点，统一24px尺寸
- **动效**：页面切换淡入，数据加载骨架屏，计时器数字翻转动效

### 4.3 响应式设计
- 采用移动优先设计，主要适配375px-428px移动端屏幕
- 卡片和列表使用弹性布局，自适应不同宽度
- 底部Tab导航固定，内容区域可滚动
- 横屏时自动调整布局为侧边导航
